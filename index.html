What is PHP Test Fixture?

PHP Test Fixture is a simple, minimalistic and efficient PHP testing framework authored and maintained by none other than DonaldKellett. It is simple in the sense that PHP Test Fixture is very simple to operate - it does not require the command line like some of the more popular PHP testing frameworks do, you just have to execute the PHP file containing your test cases using a local server (e.g. MAMP) like you would any other PHP script and ta-da - you have a visual representation of the success of your test cases!

It is minimalistic in the sense that at the time of writing, the entire test fixture only occupies 156 lines of PHP code (of course excluding the <?php ?> script tags) which means that the code for the entire test fixture can be downloaded at the blink of an eye and its execution speed is as fast as a PHP testing framework can get. Of course, there's the added benefit that being small means it would barely take any storage space in your computer/server!

It is efficient in the sense that because of its tiny file size, your test cases can be executed in the blink of an eye which can save you a lot of time if you frequently test your code.

Why choose PHP Test Fixture over other (more well-known and popular) PHP testing frameworks easily accessible all over the Internet?

Here are a few major reasons why you should seriously consider using PHP Test Fixture as your PHP testing framework for your TDD (Test-Driven Development):

  1. It is simple, minimalistic and efficient (as mentioned above)
  2. Operating it and conducting tests require nothing other than simple PHP knowledge - as long as you know PHP, you are highly unlikely to run into any technical difficulties when using PHP Test Fixture
  3. Unlike some of the more popular and well-known PHP testing frameworks, PHP Test Fixture gives you very visual yet simple feedback regarding the success of your test cases - you can immediately know if your code has passed all the test cases at the blink of an eye as soon as the page is loaded
  4. The output produced by PHP Test Fixture is HTML-friendly which means you do not have to worry about wrapping the tests in <p></p> tags or echoing line breaks as the test cases automatically do it for you

What can PHP Test Fixture be used for?

As stated above, the entire purpose of the existence of PHP Test Fixture is to make the implementation of (everyday) Test-Driven Development easier, more convenient and accessible to PHP developers of all abilities.

In most cases, TDD is usually used within a company to test their own code to see if it works properly (say, for example, their company website or mobile application). Individual developers may also use TDD as their preferred method of testing out their own code. And, of course, the vast majority of PHP testing frameworks out there help you to achieve just that.

However, with PHP Test Fixture, you can go one step further - if you have the PHP skills to implement it, you can make good use of the return values of the test cases to test other people's code. What would be the benefit of testing other people's code, you may ask. The ability to test other people's code (made possible by PHP Test Fixture) enables you (or your company if you work for one) to use the familiar TDD techniques used by your company (or yourself) to assess the (coding) ability of potential developers that you wish to work by your side to complete a major coding project, for example. While you can use services like Strive Qualified or Codewars for just that (and by the way, they are really great, you really should try them out, especially Codewars since it is free), PHP may not be supported yet in some of these services (at least at the time of writing, 10/05/16) and by using PHP Test Fixture to create your own tests for your potential candidates/co-workers, there is more flexibility as to what you can test and how you can do it (provided you have the coding skills required to implement it).

I want to see an example of the PHP Test Fixture being used and how the output looks like. Where can I find such an example?

When you download the PHP Test Fixture, it automatically comes with a Demo called demo.php. The Demo PHP file includes numerous examples of how each assertion method should be used and its output for both passing and failing tests respectively. The Demo file is best used when you put the source code of the Demo file and the HTML (visual) output side-by-side to understand what is going on.
